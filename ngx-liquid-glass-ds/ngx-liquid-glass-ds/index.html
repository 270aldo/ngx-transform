The 3-layer stack 01 / 05

Every prototype imports the layers in this order — never out of sequence. Brand tokens at the bottom, Liquid Glass materials on top, dashboard components last (optional for non-density surfaces like landing or workout active).

Layer 01 · Brand
Brand tokens
tokens-brand.css
Navy + purple + gray ramps, semantic tokens (--bg, --fg, --accent), United Sans Cond + Neue Haas Grotesk Text Pro, spacing scale, radii, shadows, easing.
  • --ngx-navy
  • --ngx-purple
  • --font-display
  • --font-text
  • --space-1..32
  • --ease-out
Layer 02 · Materials
Liquid Glass
tokens-lg.css
Glass fills, rims, purple glows, blurs. Wallpapers (soft / regular / bloom) with diamond X-strand pattern. Primitives: .lg-glass, .lg-cta, .lg-headline, .lg-phone, .lg-browser, .nudge.
  • --lg-glass-purple
  • --lg-glow-purple-md
  • --lg-blur-regular
  • .lg-wallpaper-*
  • .lg-cta
  • .nudge
Layer 03 · Components
Dashboard density
components.css
B2B + dashboard patterns: sidebar nav, KPI tiles, range pills, status pills, dense tables, bar charts, heat grids, message bubbles, program blocks.

Quickstart 02 / 05

Drop the 3 stylesheets, wrap your canvas in a wallpaper class, and you're in. tokens-brand.css + tokens-lg.css are mandatory; components.css only for dashboard / B2B density surfaces.

<!-- 1. Import all 3 layers in order --> <link rel="stylesheet" href="tokens-brand.css"> <link rel="stylesheet" href="tokens-lg.css"> <link rel="stylesheet" href="components.css"> <!-- 2. Wrap your canvas with a wallpaper --> <body data-theme="dark"> <div class="lg-wallpaper-soft">/* dashboard */</div> <div class="lg-wallpaper-regular">/* mobile · landing */</div> <div class="lg-wallpaper-bloom">/* workout · focus */</div> <!-- 3. Compose with primitives --> <h1 class="lg-headline">PUSH HARD. <span class="charge">LIVE LONG.</span></h1> <div class="ngx-kpi ngx-kpi--feat">...</div> <button class="lg-cta lg-cta--primary">Start session</button>
Glass dosage 03 / 05

Three intensities, one canonical default. Going higher reads as showroom; going lower loses the LG identity. Each surface declares its dose in the page root, optionally swappable via [data-dose="light" | "full"].

30%
Light
Headers + nav + CTAs only. Body remains flat navy. For high-density legacy migrations or accessibility-first views.
60%
Medium
Tab bar + KPI cards + nudges + sheets + sidebars. Numbers, headers and charts stay flat. The default across all 4 surfaces.
80%
Full
Everything glass. Hero / focus moments — workout active, onboarding intro, splash, marketing hero.
The 10 rules 04 / 05

Visual non-negotiables. If a design breaks one of these, it stops being NGX Liquid Glass.

01
Tabular nums on every number
No exceptions. font-variant-numeric: tabular-nums;
02
One purple charge per composition
Purple is signal — pick the CTA, the active state, OR the featured tile to wear it.
03
Glass = both filter properties
-webkit-backdrop-filter AND backdrop-filter. Safari still needs the prefix.
04
Diamond pattern visible
If you can't see the X-strand under the wallpaper, opacity is wrong.
05
Headlines: Heavy 900 · CAPS · lh ≤1
United Sans Cond, letter-spacing −0.015em to −0.025em. Logo-adjacent.
06
Labels micro · CAPS · wide tracking
10–11px · letter-spacing 0.14–0.18em. Every section gets an eyebrow.
07
Sharp corners
--radius-2xl forbidden except for the rounded PNG feature-tile icons.
08
No flat gradient backgrounds
Mesh blooms behind glass YES. Purple-to-pink fades NO.
09
Voice: data → interpretation → action
Three clauses max. Drop the small-talk. Spanish o English equally.
10
Emoji never
Lucide SVG, PNG feature tiles, or unicode glyphs (→ ↑ × ✓ • …).
